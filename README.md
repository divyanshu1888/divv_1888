#  Mini Tutorial: Git, GitHub & Markdown

##  What is Git?
Git is a **version control system**.

 In simple terms:
- It helps you **track changes** in your code
- You can go back to **previous versions**
- Multiple people can **work on the same project**

Think of Git like a **save history for your project**.

---

##  How to Push Code to GitHub

GitHub is a website where you **store your code online**.

###  Steps to push your code:

1. Open terminal in your project folder  
2. Initialize Git:
```bash
git init
```

3. Add files:
```bash
git add .
```

4. Commit changes:
```bash
git commit -m "First commit"
```

5. Connect to GitHub:
```bash
git remote add origin https://github.com/your-username/repo-name.git
```

6. Push your code:
```bash
git push -u origin main
```

Now your code is on GitHub!

##  What is Markdown?

Markdown is a **simple way to format text**.

Instead of using buttons (like bold, headings),  
you use **symbols and characters**.

It is used in:
- README files  
- Documentation  
- GitHub pages  

---

##  Basic Markdown Syntax

###  Headings
```markdown
# Heading 1
## Heading 2
### Heading 3
```

---

###  Lists

**Unordered List:**
```markdown
- Item 1
- Item 2
- Item 3
```

**Ordered List:**
```markdown
1. First
2. Second
3. Third
```

---

###  Code Blocks

Inline code:
```markdown
`print("Hello")`
```
Code block:
```markdown
```python
print("Hello World")
```
```
---
###  Links

```markdown
[Google](https://www.google.com)
```
---

##  Summary

- **Git** → Tracks changes in your code  
- **GitHub** → Stores your code online  
- **Markdown** → Formats text easily
